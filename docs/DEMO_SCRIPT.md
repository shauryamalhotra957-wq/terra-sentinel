# Demo Script

## 30-Second Opening

Terra Sentinel is a humanitarian risk command center. It takes disaster stressors like rainfall, river level, wind, heat, hospital load, road blockage, and communications outage, then converts them into district risk, lifeline failure, resource deployment, and public warning packets.

The point is not to predict the future perfectly. The point is to make response decisions inspectable, fast, and defensible when minutes matter.

## 5-Minute Walkthrough

1. Start on `Monsoon Surge`.
2. Point to the metric strip: city risk, exposed population, resource coverage, and equity guardrail.
3. Click `South Flats` on the map and show the district score, shelter demand, and primary driver.
4. Move rainfall and river level upward. The map, lifelines, forecast, and allocations update immediately.
5. Switch to `Heat Health`. Show how the risk shifts from flood-heavy districts toward dense medical-vulnerability zones.
6. Open the resource list. Explain units, ETA, people covered, and the rationale line.
7. Show the warning panel. Messages are prepared for SMS, WhatsApp, public address, and radio with checksums.
8. Export the briefing packet and explain how an emergency desk could attach it to an incident log.

## Professor Questions To Expect

**Is this real AI?**  
It is deterministic decision intelligence, not a black-box model. That is deliberate: emergency systems need explainable outputs before ML calibration.

**Where is the data from?**  
The current data is synthetic, shaped around public disaster-management frameworks. Production use would replace it with municipal GIS, census, hospital, sensor, and incident data.

**How do you avoid harm?**  
The app is explicit that it is not certified for real emergency operations. A production version would need expert validation, approval workflows, signed packets, audit logs, and governance.

**What is technically impressive?**  
It has a real domain engine, finite resource optimization, explainability, accessibility checks, exportable packets, offline support, security notes, and automated tests.
